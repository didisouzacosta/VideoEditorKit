# Save Preview Thumbnail Canvas Position Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fazer o video salvo manualmente e a thumbnail persistida refletirem a mesma geometria de canvas/posicao vista na edicao, mantendo a exportacao correta.

**Architecture:** O preview e a exportacao ja passam pelo `VideoCanvasMappingActor`; a assimetria esta no caminho de manual save, que usa `renderIntent: .saveNative`. A correcao deve manter o frame rate nativo do save, mas resolver o `renderSize` pelo canvas quando existir preset/freeform/social destination, para que `resolvedCanvasRenderRequest(... exportSize:)` use a mesma area de composicao da edicao/exportacao. A thumbnail continua derivada do arquivo salvo, mas ganha testes para posicao de canvas.

**Tech Stack:** Swift, SwiftUI-facing APIs, AVFoundation, Swift Testing, Xcode workspace `Example/VideoEditor.xcworkspace`.

## Current Diagnosis

- `VideoEditorManualSaveRenderer.Dependencies.live.renderEditedVideo` calls `VideoEditor.startRender(... renderIntent: .saveNative(sourceFrameRate: nil))`.
- `VideoEditor.resolvedRenderProfile(... intent: .saveNative)` currently returns `evenPixelSize(for: sourceSize)` even when `editingConfiguration.canvas.snapshot.preset != .original`.
- `resizeAndLayerOperation` then calls `resolvedCanvasRenderRequest(... exportSize: exportProfile.renderSize)`, so the canvas transform is applied inside the original source-sized render target instead of the preset canvas target.
- Normal export qualities use `resolvedOutputRenderSize(...)`, which already respects canvas presets, so export can look correct while saved preview/thumbnail do not.
- Existing test `resolvedSaveNativeRenderProfilePreservesSourceResolutionAndFrameRate` currently locks in the wrong behavior for preset canvas saves.

## Desired Behavior

- Manual save keeps source frame rate when available.
- Manual save uses the canvas render size whenever the edit has a non-original canvas preset, legacy freeform crop, or social destination.
- Manual save keeps source render size for an unmodified `.initial` configuration.
- Saved thumbnail generated from the saved edited video matches the saved edited video.
- Source autosave thumbnails generated by `VideoEditingThumbnailRenderer` continue to reflect `VideoCanvasSnapshot.transform.normalizedOffset`.
- `.original` export remains available and still uses prepared original/saved media only when the editing fingerprint matches.

### Task 3: Fix Native Save Canvas Render Size

**Files:**
- Modify: `Sources/VideoEditorKit/Internal/Models/Enums/VideoEditor.swift`

- [ ] **Step 1: Add a native render-size helper**

Add this private helper near `resolvedRenderProfile(...)`:

```swift
    private static func resolvedNativeRenderSize(
        for sourceSize: CGSize,
        editingConfiguration: VideoEditingConfiguration
    ) -> CGSize {
        if let canvasSize = preferredCanvasRenderSize(
            for: sourceSize,
            editingConfiguration: editingConfiguration
        ) {
            return evenPixelSize(for: canvasSize)
        }

        return evenPixelSize(for: sourceSize)
    }
```

- [ ] **Step 2: Use the helper for save-native and original export profiles**

In `resolvedRenderProfile(...)`, change the `.saveNative` and `.export(.original)` branches to:

```swift
        switch intent {
        case .saveNative(let sourceFrameRate):
            renderSize = resolvedNativeRenderSize(
                for: sourceSize,
                editingConfiguration: editingConfiguration
            )
            resolvedFrameDuration = frameDuration(forSourceFrameRate: sourceFrameRate)
            presetNames = resolvedNativeSavePresetNames(isSimulatorEnvironment: isSimulatorEnvironment)
        case .export(let videoQuality) where videoQuality.isOriginal:
            renderSize = resolvedNativeRenderSize(
                for: sourceSize,
                editingConfiguration: editingConfiguration
            )
            resolvedFrameDuration = frameDuration(forSourceFrameRate: nil)
            presetNames = resolvedNativeSavePresetNames(isSimulatorEnvironment: isSimulatorEnvironment)
        case .export(let videoQuality):
            renderSize = resolvedOutputRenderSize(
                for: sourceSize,
                editingConfiguration: editingConfiguration,
                videoQuality: videoQuality
            )
            resolvedFrameDuration = frameDuration(for: videoQuality)
            presetNames = resolvedExportPresetNames(
                for: videoQuality,
                isSimulatorEnvironment: isSimulatorEnvironment
            )
        }
```

## Risk Notes

- Changing `.saveNative` render size affects manual save and `.original` export renders that pass through native-save intent. This is intended for edited canvas configurations, but must not alter `.initial`.
- Prepared `.original` export shortcuts should remain guarded by `continuousSaveFingerprint`; stale saved videos must not be reused if the canvas transform changed.
- `preferredCanvasRenderSize(...)` currently prefers canvas size for non-original preset, social destination, or freeform crop. If a future edit supports non-identity transform with `.original` preset only, source-size output is still expected.
- Do not replace the separate preview/export engines in this fix. Keep the change local to render profile sizing and characterization tests.
